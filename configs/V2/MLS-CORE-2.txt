# MLS-CORE-2 Configuration

Device role: Secondary Multilayer Core Switch
Model: Cisco Catalyst 3560
Function: Inter-VLAN routing, HSRP gateway redundancy, Rapid PVST, DHCP relay and redundant connectivity to the Internet edge router.

MLS-CORE-2 acts as the preferred gateway and Spanning Tree root for VLANs 20, 40 and 99.

---

## Basic Configuration

```bash
hostname MLS-CORE-2

ip routing

ip domain-name SCHOOL.LOCAL

username ADMIN privilege 15 secret <SECURE_PASSWORD>
```

SSH access uses the local user database.

The current running configuration does not contain an `enable secret` command or the explicit command:

```bash
ip ssh version 2
```

---

## VLAN Configuration

The VLANs are transported through the trunk links and have Layer 3 SVI interfaces configured on the core switch.


## Rapid PVST Configuration

Rapid PVST provides faster Layer 2 convergence and prevents switching loops in the redundant topology.

MLS-CORE-2 is configured as the preferred root bridge for VLANs 20, 40 and 99.

```bash
spanning-tree mode rapid-pvst

spanning-tree vlan 20,40,99 priority 24576
spanning-tree vlan 10,30,50 priority 28672
```

This configuration distributes the Spanning Tree root role between both multilayer core switches.

## Access Switch Trunks

Each access switch is connected redundantly to both multilayer core switches.

### FastEthernet0/1 Trunk

FastEthernet0/1 currently allows all production and management VLANs.

```bash
interface FastEthernet0/1
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30,40,50,99
exit
```

## Inter-Core Trunk

FastEthernet0/6 connects MLS-CORE-2 with MLS-CORE-1.

All production and management VLANs are allowed through the inter-core trunk.

```bash
interface FastEthernet0/6
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30,40,50,99
exit
```

This trunk allows HSRP communication and maintains Layer 2 connectivity between both core switches.

---

## Routed Link to R-INTERNET

GigabitEthernet0/1 operates as a Layer 3 interface between MLS-CORE-2 and R-INTERNET.

```bash
interface GigabitEthernet0/1
 description link to R-INTERNET
 no switchport
 ip address 10.0.0.6 255.255.255.252
exit
```

Network information:

```text
Network: 10.0.0.4/30
R-INTERNET: 10.0.0.5
MLS-CORE-2: 10.0.0.6
```

## HSRP and SVI Configuration

HSRP provides a shared virtual default gateway for each VLAN.

End devices use the virtual HSRP address rather than the physical IP address of an individual core switch.

---

### VLAN 10 — ALUMNOS

MLS-CORE-1 is the preferred active HSRP router for VLAN 10.

```bash
interface Vlan10
 ip address 192.168.0.3 255.255.254.0
 ip helper-address 192.168.2.98
 standby 10 ip 192.168.0.1
 standby 10 preempt
exit
```

```text
Network: 192.168.0.0/23
HSRP virtual gateway: 192.168.0.1
MLS-CORE-1: 192.168.0.2
MLS-CORE-2: 192.168.0.3
Preferred active core: MLS-CORE-1
```

---

## DHCP Relay

The central DHCP server is located in VLAN 50.

```text
DHCP server: 192.168.2.98
```

DHCP relay is configured on the client VLAN interfaces:

```bash
interface Vlan10
 ip helper-address 192.168.2.98
exit

interface Vlan20
 ip helper-address 192.168.2.98
exit

interface Vlan30
 ip helper-address 192.168.2.98
exit

interface Vlan40
 ip helper-address 192.168.2.98
exit
```

The helper address is configured on both multilayer core switches so DHCP service remains available during a gateway failover.

## Default Route

Unknown destinations are forwarded to R-INTERNET through the dedicated MLS-CORE-2 routed link.

```bash
ip route 0.0.0.0 0.0.0.0 10.0.0.5
```

## VTY and SSH Configuration

The VTY lines use the local user database and only permit SSH as the remote management protocol.

```bash
line vty 0 4
 login local
 transport input ssh
exit

line vty 5 15
 login local
 transport input ssh
exit
```

Unlike MLS-CORE-1, the current MLS-CORE-2 configuration does not contain an `access-class` command restricting SSH access to the administrator VLAN.

## Configuration Review

The following differences should be reviewed before publishing the final repository.

### FastEthernet0/5 VLAN mismatch

MLS-CORE-2 currently contains:

```bash
interface FastEthernet0/5
 switchport trunk allowed vlan 50,99
```

MLS-CORE-1 uses FastEthernet0/5 for the administrator switch and allows:

```bash
switchport trunk allowed vlan 30,99
```

If FastEthernet0/5 connects to `SW-ADMINS`, the MLS-CORE-2 allowed VLAN list should be checked.

---

### FastEthernet0/1 VLAN difference

MLS-CORE-1 FastEthernet0/1 only allows VLANs 10 and 99.

MLS-CORE-2 FastEthernet0/1 currently allows all VLANs:

```bash
switchport trunk allowed vlan 10,20,30,40,50,99
```

This may work, but it is not consistent with the VLAN pruning applied on MLS-CORE-1.

---

### SSH access restriction

MLS-CORE-1 restricts SSH management with the `SSH_ADMINS_ONLY` ACL.

MLS-CORE-2 currently permits SSH connections through its VTY lines without an access-class restriction.

---

### SSH version

The current running configuration does not explicitly include:

```bash
ip ssh version 2
```

This should be verified before documenting SSH version 2 as implemented on MLS-CORE-2.
